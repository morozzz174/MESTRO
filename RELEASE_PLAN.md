## 🔧 Этап 2: Настройка оплаты

### 2.1 Регистрация в YooKassa
1. Перейти на [yookassa.ru](https://yookassa.ru)
2. Зарегистрироваться как ИП Морозов М.И.
3. Получить:
   - `shopId` (идентификатор магазина)
   - `secretKey` (секретный ключ API)
4. Включить «Безопасную сделку» (для приёма платежей от физлиц)
5. Настроить вебхуки: `https://your-backend.com/yookassa/webhook` (если есть сервер)

### 2.2 Настройка переменных
Создать файл `.env` в корне проекта:
```env
UCALLER_SERVICE_ID=ваш_service_id
UCALLER_SECRET_KEY=ваш_secret_key
YOOKASSA_SHOP_ID=ваш_shop_id
YOOKASSA_SECRET_KEY=ваш_secret_key
```

### 2.3 Тестирование оплаты
YooKassa предоставляет тестовый режим:
- Тестовые карты: `1111 1111 1111 1111`, срок `12/27`, CVV `123`
- В тестовом режиме деньги не списываются
- Платежи приходят в статус `succeeded`

### 2.4 ⚠️ Важное примечание
**Прямой вызов YooKassa API из мобильного приложения НЕ безопасен.** SecretKey будет виден в APK.

**Рекомендуемые варианты:**
1. **Backend-сервер** (рекомендуется): Создать простой сервер (Node.js/Python), который:
   - Принимает запрос от приложения
   - Создаёт платёж через YooKassa API
   - Возвращает URL оплаты в приложение
   - Принимает вебхук от YooKassa и активирует премиум

2. **RUstore Billing** (для RUstore): Использовать встроенную систему оплаты RUstore — она безопаснее и соответствует требованиям магазина.

3. **Демо-режим** (текущий): Пока нет сервера — оплата работает в демо-режиме (премиум активируется без реального платежа). Для MVP допустимо.


Будет запрошено:

- Имя: Морозов Максим Игоревич
- Организация: ИП Морозов
- Город: Челябинск
- Страна: RU

### 3.2 Настройка build.gradle
Создать файл `c:\mestro_2\android\key.properties`:
```properties
storePassword=ваш_пароль_keystore
keyPassword=ваш_пароль_key
keyAlias=mestro-key
storeFile=mestro-release.keystore
```

В `android/app/build.gradle.kts` добавить:
```kotlin
val keystorePropertiesFile = rootProject.file("key.properties")
val keystoreProperties = Properties()
if (keystorePropertiesFile.exists()) {
    keystoreProperties.load(keystorePropertiesFile.inputStream())
}

android {
    signingConfigs {
        create("release") {
            keyAlias = keystoreProperties["keyAlias"] as String
            keyPassword = keystoreProperties["keyPassword"] as String
            storeFile = file(keystoreProperties["storeFile"] as String)
            storePassword = keystoreProperties["storePassword"] as String
        }
    }
    buildTypes {
        release {
            signingConfig = signingConfigs.getByName("release")
            // ... остальные настройки
        }
    }
}
```

⚠️ **Добавить в `.gitignore`:**
```
android/key.properties
android/app/mestro-release.keystore
```

## 📦 Этап 4: Сборка APK/AAB

### 4.1 Release APK (для RUstore)
```bash
flutter clean
flutter pub get
flutter build apk --release
```
Файл: `build/app/outputs/flutter-apk/app-release.apk`

### 4.2 Release AAB (для Google Play / RUstore)
```bash
flutter build appbundle --release
```
Файл: `build/app/outputs/bundle/release/app-release.aab`

### 4.3 Проверка APK
```bash
# Проверка подписи
jarsigner -verify -verbose -certs build/app/outputs/flutter-apk/app-release.apk

# Размер APK (должен быть < 150 МБ для RUstore)
ls -lh build/app/outputs/flutter-apk/app-release.apk
```
